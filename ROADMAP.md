# AthLink Hub Roadmap

## Phase 1: Static MVP Prototype

Goal: Create a convincing web app prototype that shows how AthLink Hub works before adding real database/auth.

### Must Have
- Homepage with clear Atlantic Canada positioning
- Guided search by sport, age group, location, and need
- Search results page
- Provider cards
- Provider profile page
- Saved providers page
- Add business page
- Claim listing flow
- Admin review dashboard
- Sample provider data for Newfoundland & Nova Scotia

### Not Yet
- Payments
- Real booking system
- Reviews
- Messaging
- Mobile app store release
- Full AI scraping pipeline

---

## Phase 2: Database + Admin Workflow

Goal: Move sample provider data into a real database.

### Must Have
- Supabase database
- Provider table
- Sports table
- Categories/needs table
- Business claim status
- Admin review status
- Source tracking
- Manual add/edit provider workflow

---

## Phase 3: AI Discovery Pipeline

Goal: Use AI/tools to find and enrich provider listings.

### Must Have
- Discovery workflow
- Source URLs
- Confidence score
- Duplicate detection
- Human review before publishing
- Admin approval queue

---

## Phase 4: Provider Monetization

Goal: Let businesses claim, improve, and promote listings.

### Possible Revenue
- Featured listings
- Paid provider profiles
- Lead generation
- Booking fee later
- Subscription plan later

---

## Phase 5: Booking

Goal: Add in-app booking once provider types and workflows are understood.

### Must Have Later
- Provider availability
- Session types
- Booking requests
- Notifications
- Payments
- Cancellation rules
